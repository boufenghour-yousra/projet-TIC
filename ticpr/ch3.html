chapitre 3:Le Modèle OSI

1) Les Composants matériel d’un Réseau informatique :

    Les cartes réseau :

    Les cartes réseau sont des composants matériels qui permettent à un ordinateur de se connecter à un réseau .Elles jouent un rôle essentiel dans la communication entre ordinateurs car elles permettent a l’ordinateur de transmettre et de recevoir des données via des protocoles de communication spécifiques

    SWITCH :

    C’est un composant matériel utilisé pour connecter de appareils sur un réseau local .Le switch permet à plusieurs appareils à communiquer entre eux en utilisant des adresses MAC . Les switches sont disponibles en différentes tailles ,allant des petits switches pour les réseaux domestiques etc. , aux switches de niveau entreprise pour les réseaux plus importants

    HUB :

    Un hub est un composant matériel utilisé pour connecter plusieurs appareils sur un réseau local .Il fonctionne comme un répéteur. Contrairement à un switch qui utilise une table de correspondance pour envoyer les données seulement à l’appareil destinataire, le hub ne possède pas cette fonctionnalité.

2) Le Modèle OSI :

OSI :

C’est un modèle de référence pour les communications de réseau qui divise les communications en sept couches . Et chaque couche est responsable de fonctions spécifiques liées à la communication de données .

La responsabilité de chaque couche :

La couche physique :

Elle est responsable de la transmission des données sur le support physique

La couche liaison de données:

Elle est responsable de la transmission des données entre les appareils qui partagent le même support physique

La couche réseau:

Elle est responsable du routage des données à travers le réseau

La couche transport:

Elle est responsable de l’acheminement des données entre les appareils

La couche session :

Elle est responsable de l’établissement et de la gestion des sessions entre les appareils

La couche présentation :

Elle est responsable de la conversion des données dans un format qui peut être compris par l’appareil de destination

La couche application :

Elle est responsable de la communication entre les applications sur les appareils de réseau

5) L’adresse IPV4

- Chaque adresse IPV4 contient deux parties :

    ° La première partie des bits de valeur supérieur : Adresse réseau.

    ° La deuxième partie des bits : Adresse hôte.

- Quand la valeur des bits de la partie réseau est la même on dit que ce réseau est un hôte.

- Le nombre de bits contenus dans la partie hôte détermine le nombre de machines possible sur ce réseau .

- Les différents type d’adresse IPV4 :

    1- L’adresse réseau :

    Tous les bits de la partie hôte =0

    2- L’adresse de diffusion :

    C’est une adresse dans laquelle tous les bits de la partie hôte =1 .

    3- L’adresse hôte :

    On attribue les valeurs situés entre l’adresse réseau et l’adresse de diffusion sur machines situés sur ce réseau .

    4- Le masque de réseau et de sous réseaux :

    Tous les bits de la partie réseaux =1
